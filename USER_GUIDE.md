# 📖 用户使用手册

## 📥 安装与启动

### Windows 用户（推荐）

1. **下载软件**
   - 访问 [GitHub Releases](https://github.com/your-username/file-converter/releases/latest)
   - 下载 `文件转换器.exe`

2. **运行程序**
   - 双击 `文件转换器.exe` 即可启动
   - 无需安装，开箱即用
   - 首次运行可能被 Windows Defender 拦截，点击"仍要运行"

### 从源码运行（开发者）

```bash
# 克隆仓库
git clone https://github.com/your-username/file-converter.git
cd file-converter

# 安装依赖
pip install -r requirements.txt

# 运行程序
python md_converter.py
```

## 🎯 基础操作

### 1️⃣ 选择源文件

#### 方法一：点击按钮选择

1. 点击界面左侧的**「选择文件」**按钮
2. 在弹出的文件选择窗口中，选择要转换的文件
3. 支持的文件类型：
   - `.md` / `.markdown` - Markdown 文件
   - `.html` / `.htm` - HTML 网页
   - `.docx` - Word 文档
   - `.pdf` - PDF 文件
   - `.txt` - 纯文本

#### 方法二：拖拽文件（更快捷！）

1. 直接从文件夹拖拽文件
2. 拖到软件窗口任意位置
3. 松开鼠标即可自动加载

### 2️⃣ 选择输出格式

软件界面中间显示 5 个格式按钮：

- **PDF** - 便携文档格式（红色）
- **Word** - Microsoft Word 文档（蓝色）
- **HTML** - 网页文件（橙色）
- **Markdown** - 纯文本标记（绿色）
- **TXT** - 纯文本（灰色）

#### 操作说明

- **点击**任意格式按钮来选择输出格式
- **可以多选**：同时选择多个格式进行批量转换
- **灰色禁用**：表示与源文件格式相同，无需转换

**示例**：
- 选择一个 `.md` 文件
- 同时选中 PDF、Word、HTML 三个格式
- 一次性生成 3 个不同格式的文件

### 3️⃣ 设置保存位置

#### 默认位置

软件默认将转换后的文件保存在**源文件所在目录**。

例如：
- 源文件：`C:\Documents\我的文档.md`
- 输出文件：`C:\Documents\我的文档.pdf`

#### 自定义位置

1. 点击**「更改」**按钮
2. 选择目标文件夹
3. 所有转换后的文件将保存到该位置

### 4️⃣ 开始转换

1. 确认右侧**「当前任务」**面板显示正确
   - 文件名
   - 输出格式
   - 保存位置

2. 点击底部的**「开始转换」**按钮

3. 进度条显示转换进度

4. 转换完成后，弹出结果窗口

## 🔄 转换场景示例

### 场景 1：Markdown 转 PDF（写作发布）

**用途**：将 Markdown 笔记导出为可打印的 PDF

1. 选择 `.md` 文件
2. 勾选 **PDF** 格式
3. 点击「开始转换」
4. 获得格式化美观的 PDF 文档

**特点**：
- 自动渲染代码高亮
- 表格格式完整保留
- 图片自动嵌入

### 场景 2：Word 转 PDF（文档分享）

**用途**：将 Word 文档转为不可编辑的 PDF

1. 选择 `.docx` 文件
2. 勾选 **PDF** 格式
3. 转换完成

**优势**：
- 使用 Microsoft Word 引擎
- 完美保留所有格式
- 字体、样式、排版完全一致

### 场景 3：HTML 转 Markdown（网页保存）

**用途**：将网页内容保存为可编辑的 Markdown

1. 保存网页为 `.html` 文件
2. 选择该文件
3. 勾选 **Markdown** 格式
4. 获得纯文本标记文件

**用途**：
- 网页文章归档
- 博客内容迁移
- 笔记整理

### 场景 4：PDF 转 Word（内容编辑）

**用途**：将 PDF 转为可编辑的 Word 文档

1. 选择 `.pdf` 文件
2. 勾选 **Word** 格式
3. 转换（耗时较长）

**注意**：
- 需要安装 Microsoft Office
- 复杂排版可能有偏差
- 扫描版 PDF 无法识别文字

### 场景 5：多格式批量输出

**用途**：一次生成多种格式用于不同场景

1. 选择源文件（如 `.md`）
2. **同时勾选** PDF、Word、HTML
3. 一键转换
4. 获得 3 个不同格式的文件

**应用**：
- 文档发布（PDF 分享 + Word 编辑）
- 博客发布（HTML 上传 + Markdown 备份）
- 多平台适配

## ⚙️ 高级功能

### 文件冲突处理

当目标文件夹已存在同名文件时，软件提供 4 种处理方式：

#### 1. 智能重命名（推荐）

- 自动添加序号避免冲突
- 例如：`文档.pdf` → `文档_1.pdf`
- 原文件保持不变

#### 2. 覆盖替换

- 直接覆盖同名文件
- **警告**：原文件将被删除，无法恢复
- 适用于更新旧版本

#### 3. 跳过

- 不转换该格式
- 其他格式继续转换
- 适用于部分格式已存在

#### 4. 取消

- 取消整个转换任务
- 不生成任何文件
- 重新调整设置

### 拖拽快速转换

**快捷操作**：

1. 将文件拖入窗口
2. 文件信息自动加载
3. 上次选择的格式自动保留
4. 点击「开始转换」即可

**效率提升**：
- 无需点击「选择文件」
- 批量处理多个文件时更快捷

### 自动更新检查

#### 启动时自动检查

- 程序启动时在后台检查更新
- 如有新版本，弹窗提示
- 点击「立即下载」打开下载页面

#### 手动检查

1. 菜单栏 → **帮助** → **检查更新**
2. 显示当前版本和最新版本
3. 如需更新，点击下载

## 💡 使用技巧

### 提高转换质量

1. **源文件格式优先**
   - Markdown 和 Word 转换效果最佳
   - 避免 PDF → 其他格式（信息丢失）

2. **图片处理**
   - 确保图片路径正确
   - 相对路径优于绝对路径
   - 图片文件与文档在同一目录

3. **复杂表格**
   - Markdown 表格保持简单
   - Word 复杂表格转 PDF 效果最佳

### 提高转换速度

1. **减少并发格式**
   - 一次转换 2-3 种格式为宜
   - 避免同时转换 5 种格式

2. **避免大文件**
   - PDF 文件建议 < 50MB
   - Word 文档建议 < 20MB

3. **关闭其他程序**
   - PDF 转换需要启动 Word
   - 确保 Word 未被占用

### 批量处理建议

虽然当前版本暂不支持批量文件，但可以：

1. **快速切换**
   - 拖入第一个文件 → 转换
   - 拖入第二个文件 → 转换
   - 格式选择自动保留

2. **统一输出位置**
   - 设置一次保存位置
   - 后续文件自动使用该位置

## ❓ 常见问题解答

### Q1: 转换失败怎么办？

**可能原因**：
- 文件损坏或格式错误
- 缺少必要的依赖（如 Microsoft Office）
- 文件被其他程序占用

**解决方法**：
1. 检查源文件能否正常打开
2. 确认已安装 Microsoft Office（PDF 转换需要）
3. 关闭占用文件的程序
4. 尝试其他格式转换路径

### Q2: PDF 转换显示错误？

**情况 A**：提示"无法启动 Word"

- **原因**：未安装 Microsoft Office
- **解决**：安装完整版 Office（不支持 WPS）

**情况 B**：转换后格式错乱

- **原因**：PDF 结构复杂
- **解决**：尝试先转为 Word，手动调整后再转其他格式

**情况 C**：扫描版 PDF 无内容

- **原因**：PDF 是图片，无文字信息
- **解决**：使用 OCR 工具先识别文字

### Q3: Markdown 转换后代码高亮丢失？

**原因**：不同格式支持的样式不同

**建议**：
- 转 HTML 保留最完整
- 转 PDF 次之
- 转 Word 可能样式简化

### Q4: 转换速度很慢？

**影响因素**：
- 文件大小（PDF > 10MB 较慢）
- 图片数量（多图片耗时）
- 系统性能

**优化建议**：
- 分批转换大文件
- 压缩过大的图片
- 关闭不必要的后台程序

### Q5: 能否转换加密的 PDF？

**答案**：不支持

- 加密 PDF 需要先解密
- 建议使用专业 PDF 工具先解密
