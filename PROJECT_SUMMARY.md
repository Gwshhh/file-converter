## 🔧 技术实现

### 自动更新模块（updater.py）

**核心函数**：

1. `check_for_updates()` - 检查更新
   - 访问 GitHub API
   - 解析最新版本信息
   - 比较版本号
   - 返回更新状态

2. `open_download_page()` - 打开下载页面
   - 使用系统默认浏览器

3. `get_current_version()` - 获取当前版本
   - 返回版本号字符串

**依赖库**：
- `urllib.request` - HTTP 请求
- `json` - JSON 解析
- `packaging.version` - 版本号比较
- `webbrowser` - 浏览器操作

### 主程序集成（md_converter.py）

**新增类**：

1. `UpdateAvailableEvent` - 自定义更新事件
   - 用于跨线程通知

2. `UpdateDialog` - 更新提示对话框
   - 显示版本号
   - 显示更新说明
   - 提供下载按钮

**新增方法**：

1. `setup_menu()` - 设置菜单栏
2. `check_updates_on_startup()` - 启动时检查更新
3. `manual_check_updates()` - 手动检查更新
4. `show_update_dialog()` - 显示更新对话框
5. `show_about()` - 显示关于对话框
6. `customEvent()` - 处理自定义事件

## ⚠️ 重要提醒

### 发布前必须操作

1. **修改仓库配置**（updater.py 第 9 行）：
   ```python
   GITHUB_REPO = "your-username/file-converter"  # 改为实际用户名
   ```

2. **创建 GitHub 仓库**：
   - 仓库名：`file-converter`
   - 可见性：Public

3. **推送代码到 GitHub**：
   ```bash
   git remote add origin https://github.com/YOUR-USERNAME/file-converter.git
   git branch -M main
   git push -u origin main
   ```

4. **打包并发布 Release**：
   ```bash
   pyinstaller 文件转换器.spec
   # 然后在 GitHub 创建 v1.0.0 Release 并上传 exe
   ```
